Skills don't execute everything inline — they deploy specialized subagents, each receiving only the context it needs and writing only to files it owns. 20 of them: 7 framework-domain agents that ship inside the pack, plus 13 dev-engineer agents used during active development. Input isolation is structural — by design, no agent sees another's output within the same review pass.Skill không chạy hết mọi thứ inline — chúng triển khai subagent chuyên biệt, mỗi cái chỉ nhận context cần thiết và chỉ ghi vào file nó sở hữu. 20 agent: 7 agent domain-framework ship cùng gói, cộng 13 agent dev-engineer dùng trong phát triển chủ động. Anti-anchoring: structural input isolationChống neo: cô lập đầu vào cấu trúc

The most dangerous failure mode in multi-agent review is anchoring — where one agent's verdict primes the next. Two ORC skills enforce structural prevention rather than relying on prompt instructions.Lỗi nguy hiểm nhất trong review đa-agent là neo — khi phán quyết của agent này dẫn dắt agent kế. Hai skill ORC ép ngăn chặn cấu trúc thay vì dựa vào chỉ dẫn prompt.

orc:council — 4-voice framework

Frames a decision through four independent voices (Analyst · Devil's Advocate · Pragmatist · Synthesizer) that each receive only the raw question and original evidence — never each other's output. The Synthesizer reads all four only at the merge step. This prevents a single confident voice from anchoring the result.Đặt khung quyết định qua bốn giọng độc lập (Phân tích · Phản biện · Thực dụng · Tổng hợp) mỗi cái chỉ nhận câu hỏi thô và bằng chứng gốc — không bao giờ thấy đầu ra của nhau. Tổng hợp đọc cả bốn chỉ ở bước gộp. Điều này ngăn một giọng tự tin đơn neo kết quả.

orc:decisions→ 4 isolated voices

orc:santa — dual-reviewer gate

Runs two independent reviewers (Maker · Checker) in sequence: the Maker drafts a score; the Checker sees only the original artefact and pre-check criteria — never the Maker's score. Divergence above a threshold escalates to the user. The pattern is directly analogous to a two-person rule for high-stakes decisions.Chạy hai reviewer độc lập (Người làm · Người kiểm) theo thứ tự: Người làm phác điểm; Người kiểm chỉ thấy artefact gốc và tiêu chí kiểm trước — không bao giờ thấy điểm của Người làm. Phân kỳ vượt ngưỡng thì leo thang lên người dùng. Mẫu này tương tự trực tiếp quy tắc hai người cho quyết định quan trọng.

How isolation is enforced: each subagent receives a freshly-scoped prompt containing only the target artefact + its pre-check criteria. No prior agent's output is passed in the context window. The controlling skill withholds intermediate results until all independent agents have responded — only then does it synthesize. This is structural isolation, not prompt instruction. Cách cô lập được ép: mỗi subagent nhận prompt có phạm vi mới chứa chỉ artefact đích + tiêu chí kiểm trước. Không có đầu ra của agent trước nào được truyền vào cửa sổ context. Skill điều khiển giữ lại kết quả trung gian đến khi tất cả agent độc lập đã trả lời — chỉ sau đó mới tổng hợp. Đây là cô lập cấu trúc, không phải chỉ dẫn prompt.

Per-agent persistent memory: orc:agent-memoryBộ nhớ lâu dài theo agent: orc:agent-memory

Each agent accumulates calibration memory across sessions — not per-session scratchpad, but cross-session learnings stored in atomic JSONL via instinct_store.py. When an agent is deployed, orc:agent-memory loads its prior findings and feeds them as calibrated priors, preventing the same reasoning from being re-derived from scratch every invocation.Mỗi agent tích lũy bộ nhớ hiệu chỉnh xuyên phiên — không phải nháp theo phiên, mà là học từ nhiều phiên lưu dạng JSONL atomic qua instinct_store.py. Khi agent được triển khai, orc:agent-memory nạp các phát hiện trước và đưa vào như prior đã hiệu chỉnh, ngăn cùng một lập luận phải suy lại từ đầu mỗi lần gọi.

Why is input isolation structural rather than prompt-based?Tại sao cô lập đầu vào là cấu trúc thay vì dựa vào prompt?
Prompt instructions can be overridden by sufficiently detailed context bleeding into the system prompt. Structural isolation — where the prior agent's output is physically absent from the next agent's context window — cannot be "forgotten." orc:council and orc:santa enforce this by withholding intermediate results until all independent passes complete.Chỉ dẫn prompt có thể bị ghi đè bởi context đủ chi tiết rỉ vào system prompt. Cô lập cấu trúc — khi đầu ra của agent trước vật lý vắng mặt trong cửa sổ context của agent kế — không thể bị "quên." orc:council và orc:santa ép điều này bằng cách giữ lại kết quả trung gian cho đến khi tất cả lượt độc lập hoàn thành.
